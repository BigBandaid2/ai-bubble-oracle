About the AI Bubble Oracle

AI bubble, AI bubble... but what does it actually mean, and when should I care?

What is this?

I googled "when will the AI bubble pop", got a lot of alarmist click-bait, then eventually found this Polymarket bet, "AI bubble burst in 2026?". I was drawn to its clear codification of terms. Six specific conditions, TRUE or FALSE, and a number for odds. There's a proper answer! Though we'll see if it's the right one... That gave me a kernel to anchor my thinking and try to build a tracker for this wild thing we're all living through. I'm not a gambling man, I just want to know how much longer I can blissfully sleep before everything goes to shit.

So the Oracle takes that codified definition and tracks, from public data, exactly where we stand against it in real time. Each condition (NVIDIA and semiconductor drawdowns, a major supplier collapse, an OpenAI/Anthropic bankruptcy or acquisition, H100 rental prices cratering) gets its own live chart, and the contract's "3 of 6 within 90 days" rule is evaluated in front of you.

It is deliberately transparent about how it decides. The Data Sources section documents every feed, the processing pipeline that normalizes it, and the ambiguities and options baked into the rules (what counts as an all-time high, how the 90-day window is read, which H100 index applies), so you can see how each interpretation changes the answer.

Where's it going?

Tracking a single market's resolution conditions is the starting point, not the destination. The larger goal is to build a living dashboard of the data sources and indicators that actually help anticipate an AI downturn, well beyond one market's rulebook.

  • Wire up the conditions that are still manual today: bankruptcy dockets and acquisition news.
  • Add leading indicators (compute prices, capex and datacenter buildout, funding rounds, model-economics signals) that may foreshadow a turn before the headline conditions trip.
  • Make every indicator inspectable and every assumption a toggle, so you can see how it reaches each number.

How's it built?

Full disclosure, I don't actually know how to do anything. I'm just trying to figure out this vibe-coding thing like the rest of you. I had a long quiet weekend while my wife and kid were out of town and bam, here we are. This was built almost entirely with Claude Code, switching between Opus for the heavy lifting and Fable for fast iteration.

The stack is deliberately lightweight:

  • Python (standard library only, zero dependencies) generates the site, hand-written HTML, CSS, and vanilla JavaScript, with every chart drawn on an HTML <canvas> (no charting library, no front-end framework).
  • SQLite as the working data store, rebuilt from committed CSVs on each run.
  • Free public data feeds: Yahoo Finance (equity prices), Polymarket's CLOB API (the market's own odds), CourtListener / RECAP (bankruptcy dockets), GDELT (a news-volume signal), and Vast.ai (an H100 rental-price proxy).
  • Claude Haiku classifies which news articles genuinely concern a company's financial distress, keeping the buzz markers honest.
  • GitHub Actions runs the data refresh on a daily schedule; Vercel hosts the static site and auto-deploys on every push, with Vercel Web Analytics for traffic.

Open source

The whole thing, data collectors, the evaluation engine, and the site generators, is public. Fork it, check the methodology, or open an issue.

I'm totally new to open source, but I welcome ideas and collaboration. It's a fun topic, assuming this is also your idea of fun.
